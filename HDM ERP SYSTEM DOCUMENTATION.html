1. System Overview & Architecture

HDM ERP is a multi-tenant enterprise resource planning platform built on the MERN stack (MongoDB, Express, React, Node.js) with Vite as the build tool. It serves three frontend applications from a single API server.

2. Multi-Tenant Isolation

Every tenant document in the database includes a tenantId field. The tenantResolver middleware extracts the tenant ID from the JWT token or subdomain. All database queries are automatically scoped to the current tenant.

// Example: Fetching invoices for current tenant
const invoices = await Invoice.find({ tenantId: req.tenantId });

System collections (admins, plans, tenants, etc.) do not have a tenantId and are only accessible by super admins.

Critical: Never run a query without the tenantId filter on tenant collections. The tenantIsolation middleware plugin auto-adds it to all Mongoose queries.

3. Authentication & Authorization

3.1 JWT Token Flow

POST /api/public/auth/login
Body: { "email": "user@company.com", "password": "***" }
Response: { "accessToken": "eyJ...", "refreshToken": "eyJ...", "user": {...} }

POST /api/public/auth/refresh
Body: { "refreshToken": "eyJ..." }
Response: { "accessToken": "eyJ..." }

Access tokens expire in 7 days. Refresh tokens expire in 30 days. The frontend automatically refreshes tokens on 401 responses.

3.2 Device Activation

New devices must be activated with a license key before login. The activation endpoint validates the license key and trusts the device.

POST /api/public/activation
Body: { "licenseKey": "HDM-XXXX-XXXX-XXXX" }
Response: { "success": true, "message": "Device activated" }

4.4 Outward API (External Access)

External applications authenticate with an x-api-key header (not JWT). Keys are generated in Settings → API Keys.

GET/POST /api/tenant/ai/outward
Header: x-api-key: hdm_sk_xxxxxxxx

POST /api/tenant/ai/outward/query
Body: { "question": "What is my revenue?" }
Response: { "reply": "...", "raw_data": { "invoices": [...], "summary": {...} } }

POST /api/tenant/ai/outward/data
Body: { "question": "sync all data" }
Response: { "invoices": [...], "products": [...], ... }

6. AI Integration

6.1 Internal AI (Sparkle Button)

Tenants click the sparkle button in the dashboard. The query goes to POST /api/tenant/ai/query, which calls aiGatewayService.tenantQuery(). This gathers business data based on the question and module scopes, then sends it to the HDM AI endpoint.

Payload sent to AI:
{
  "query": "What is my revenue?",
  "tenant_id": "...",
  "context": { "source": "tenant", "tenant_info": {...} },
  "data": {
    "invoices": [...],
    "summary": { "totalRevenue": 38300, ... }
  }
}

6.2 Outward API Keys

Tenants generate API keys from Settings → API Keys. External apps use the x-api-key header to access /api/tenant/ai/outward/*. The middleware apiKeyAuth validates the key, sets req.tenantId and req.apiKeyScopes.

6.3 Landing Page Chatbot

The public chatbot at POST /api/public/chatbot uses the landing page config (payment methods, pricing, features) to answer visitor questions. It does not access any tenant data.

6.4 Data Exposure Details

The buildContextData function in aiGatewayService.js gathers data based on the question keywords and module scopes. Finance data is always included if scoped. Other modules are included based on keyword matching or if the query is a general business overview.

All monetary values include both raw numbers and _formatted versions in the tenant's currency (e.g., "totalRevenue": 38300, "totalRevenue_formatted": "KSh 38,300").

7. Deployment Guide

7.1 Server Deployment

# Clone repository
git clone https://github.com/Davismcintyre5/HDM-ERP.git
cd HDM-ERP/server

# Install dependencies
npm install

# Configure environment
cp .env.example .env
# Edit .env with production values

# Start with PM2
npm install -g pm2
pm2 start server.js --name hdm-server
pm2 save
pm2 startup

7.2 Client Deployment

cd HDM-ERP/client
npm install
# Set VITE_API_URL in .env.production
npm run build
# Deploy dist/ folder to static hosting (Vercel, Netlify, or nginx)

7.3 Desktop App Build

cd HDM-ERP/desktop
npm install
npm run build      # Builds client and copies to desktop/build
npm run pack:win   # Creates .exe installer in desktop/release/

9. Security

  • JWT Authentication: All tenant and admin routes require valid tokens. Tokens are verified on every request.
  • Tenant Isolation: Every query is scoped by tenantId. Cross-tenant access is impossible.
  • Rate Limiting: Public endpoints (login, chatbot) are rate-limited to prevent abuse.
  • Helmet: Security headers are set on all responses.
  • CORS: Only configured origins can access the API.
  • Password Hashing: All passwords are hashed with bcrypt (12 rounds).
  • API Key Encryption: Outward API keys and AI keys are stored securely.
  • Device Trust: New devices require license key activation before login.

10. Maintenance & Backups

Maintenance Mode: Super admins can toggle maintenance mode from Settings → General. When enabled, all public and tenant routes return a 503 response. Admin routes remain accessible.

Backups: Automatic backups run on a configurable schedule (daily, weekly, monthly). Backups are stored in server/backups/. System backups include all system collections. Tenant backups include all tenant-specific collections. Old backups are automatically cleaned up based on retention settings.

# Manual backup via admin script
node server/scripts/admin.js
# Select option 5 (Run backup now)
